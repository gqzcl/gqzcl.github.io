Linux中MySQL无法登录的解决方案

发表于 2018-11-29
· 次浏览
阅读本文可能花费您 2 分钟

前言

问题

  • 在Linux上使用MySQL遇到了一些问题，总而言之就是无法登录，报错信息就长这样： 

    Access denied for user 'root'@'localhost' (using password: NO)

  • 然后还有那种需要输入密码但是你却不知道密码是啥的也可以用这种方法。

  • 然后尝试了各种办法，找到了一个解决方案。

  • 网上还有不少方法，不过我用不了就是了。

解决方案

  1. 用vim打开在/etc/mysql/my.cnf中添加这样一句：

    [mysqld]
    skip-grant-tables

  2. 这样做是为了进入mysql控制台而不需要密码。然后重启:
    sudo service mysql restart

  3. 进入mysql：
    mysql -u root

  4. 使用以下命令重置用户密码，这里密码设置成了root，可以改成别的，随意:
    UPDATE mysql.user SET authentication_string=PASSWORD('root'), plugin='mysql_native_password' WHERE User='root' AND Host='localhost';FLUSH PRIVILEGES;

  5. 输入\q退出，再次打开/etc/mysql/my.cnf,将刚刚添加的语句删除，然后再次重启：
    sudo service mysql restart

  6. 输入mysql -u root -p登录就好了。

结语

  • 如果不行的话那就再另寻良方吧。
